**CS 360 Mobile App Portfolio Reflection**

**App Summary**

My application is called an Inventory Tracker app. One of the primary purposes of the application is to allow its users to manage their inventory in one convenient place. They can sign in, see their inventory, add new items to it, edit the quantity of certain products, and remove those which are no longer required. In addition to this, there are SMS notifications that remind users about their low inventory. My intention in designing this application was to make it simple and easy to use for inventory management.

**User Interface Design**

To satisfy users' requirements, I developed such screens as Login Screen, Registration Screen, Inventory Screen, and Settings Screen. With Inventory Screen users can easily perform adding, editing, and deleting inventory. My intention in designing the interface was to keep it neat and understandable through usage of proper labeling, buttons, and organization of the layout. I think that the interface design was successful in satisfying requirements since it paid attention to simplicity and efficiency of actions.

**Coding Process**

The coding process entailed building the application in stages, where after creating the user interface, Java coding was done to implement the features within the application. Each feature was then individually tested to ensure that there were no problems before proceeding with the next features in the app. The approach ensured that problems were detected at an early stage and could be solved. The approach can be used for software development in future projects as it ensures organization of the project.

**Testing**

Testing involved evaluating whether the user could successfully log in, create, edit, and delete items, switch between screens in the app and test the SMS notification feature. Testing was carried out through the Android emulator in Android Studio. Testing is crucial during development since bugs are identified before releasing the app to end-users. During testing, some small problems were discovered which I managed to solve.

**Challenges and Problem Solving**

A challenge I encountered was making sure that the interface of the app was connected to its functions. Ensuring the proper operation of all buttons and screens requires time and lots of tests. The challenges were solved by testing every aspect separately, finding and fixing mistakes in my code until everything worked well. It taught me the significance of patience and problem-solving skills when working on the mobile app.

**Strongest Component**

The component of the app which I find the most successful is the inventory management module. All items can be added, edited, updated and deleted from the inventory by users which satisfies the primary purpose of the application. Finishing this task helped me prove my knowledge about Android application development, interface designing, Java programming and testing.
